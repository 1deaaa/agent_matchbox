# 火柴网关贡献指南（简体中文）

## 1. 目标
本指南适用于 server/llm/agen_matchbox 子项目。目标是保证网关在多用户、配额、密钥安全和 Agent 兼容性上保持一致。

## 2. 关键原则
- 运行时权威数据源是数据库，YAML 主要用于初始化/导出。
- 不破坏 `initialize_matchbox(...) -> matchbox() -> get_user_llm(...)` 统一调用链。
- 配额口径 `sys_paid` / `self_paid` 必须保持清晰分离。
- 禁止提交明文 API Key、.env、私密配置。

## 3. 代码改动建议
- 优先在 manager.py + mixin 层扩展，不要在业务路由到处复制配置逻辑。
- 修改模型后，遵循主项目迁移流程生成迁移。
- GUI 与后端接口语义应保持一致（平台、模型、用途槽、限额策略）。

## 4. 与 Agent 生态的契合
- 该网关是 Agent 运行底座，改动时需考虑 LangChain/LangGraph 兼容。
- 不应破坏工具调用、流式输出、reasoning 字段兼容处理。
- 若调整默认行为，请同步更新 README 与调用示例。

## 5. 提交前检查
- 密钥与隐私数据是否全部排除在提交之外。
- 是否影响现有用户模型选择/用途槽位。
- 是否影响配额统计和扣费路径。
- 是否补充了必要文档说明。
