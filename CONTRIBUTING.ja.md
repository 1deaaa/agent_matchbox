# Matchbox Gateway コントリビューションガイド（日本語）

## 1. 目的
本ガイドは server/llm/agen_matchbox サブプロジェクト向けです。多ユーザー運用、配額制御、鍵管理、安全性、Agent 互換性を安定維持します。

## 2. 基本原則
- 実行時の正本はデータベース。YAML は初期化/エクスポート用途。
- 統一呼び出しチェーン `initialize_matchbox(...) -> matchbox() -> get_user_llm(...)` を維持。
- 配額口径 `sys_paid` と `self_paid` を厳密に分離。
- 平文 API Key、.env、機密設定をコミットしない。

## 3. 推奨変更パターン
- ルート層への重複実装より manager.py + mixin 拡張を優先。
- モデル変更時はメインプロジェクトの migration 手順に従う。
- GUI とバックエンド API の語義（プラットフォーム、モデル、用途スロット、配額設定）を揃える。

## 4. Agent 互換性
- 本ゲートウェイは Agent 実行基盤。LangChain/LangGraph 互換を崩さない。
- ツール呼び出し、ストリーミング、reasoning 正規化を壊さない。
- 既定動作を変更した場合は README と使用例を同時更新。

## 5. PR 前チェック
- 鍵・機密情報が混入していないか。
- ユーザーのモデル選択/用途スロット挙動に破壊的変更がないか。
- 配額集計と課金ルートが維持されているか。
- 関連ドキュメントを更新したか。
